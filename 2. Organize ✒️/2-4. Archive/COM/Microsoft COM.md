# COM이란?
> "Component Object Model" 의 약자

- 여러 컴포넌트 객체를 이용해 프로그램을 개발하는 모델
- 프로그램을 개발할 때 **Component Object** 단위로 개발하므로 더 쉽고 빠르다.
- 객체지향 개념과 비슷하지만 차이가 있다.
- *주요 차이점*
	- 프로그래밍 언어와 상관없이 객체에 접근하는 것이 가능하다.
	- 예를 들어 C++로 개발된 객체도 파이썬에서 접근이 가능하다.
